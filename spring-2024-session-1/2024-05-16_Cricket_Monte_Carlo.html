Motivation

Complex events (e.g. winning a baseball game) are composed of simple events (e.g., getting a base hit). Sometimes finding a closed form solution for a complex event is impractical or very hard.

Take the game Borel, for instance. It has a collection of experiments near a 50/50 chance for players to predict. For example:

Experiment 001: Roll d6, d6, d6, d6, d10, and d30. Will you roll 5 or more unique numbers?

Borel

finding the distribution of the number of uniques for identical dice isn’t hard.

(e.g. Pr(3 or more uniques) = 6/6 * 5/6 * 4/6 * 6/6)

But once you introduce two additional dice of different numbers of sides, the computation gets too complicated for a casual tabletop game. That’s the point though, if you enjoy math puzzles, or reading chance magazine or anything by Martin Gardner, then the experiments are deep enough to have nerd gravity. If you just want to play a short casual game with dice and more variety than Yahtzee, then you still have a shot at winning with thinking too hard.

You’re not supposed to prepare, it sort of ruins the game. You can’t have “Ruin” without “R”.

First, let’s make a function to roll dice. Not a perfectly efficient one, but a readable, informative one.

random.dice = function(dice_used = c(6,6,6,6,10,20,30), Nrolls=100000, seed=NA)
{
if(!is.na(seed)){set.seed(seed)}
    
Ndice = length(dice_used)
rollmat = matrix(NA, nrow=Nrolls, ncol=Ndice)
    
for(k in 1:Ndice)
{
rollmat[,k] = sample(1:dice_used[k], size=Nrolls, replace=T)
}

return(rollmat)
}

A lot of these experiments can be solved the same way: Generate some rolls, and apply a function row-by-row to check of each row would be a ‘success’ or not. What changes each time are the generator inputs and the function being applied.

In this first experiment, we want to know the number of unique numbers from each roll of the experiment.

count.uniques = function(roll)
{
    output = length(unique(roll))
}

Nrolls = 100000
rollmat = random.dice(dice_used = c(6,6,6,6,10,30), Nrolls=Nrolls, seed=12345)
Nuniques = apply(rollmat, 1, count.uniques)

This is Monte Carlo

So what have we done here?

Found the probabilities of small elements. (e.g. single die rolls).

Generate the results of the complex event and record.

Repeat step 2 thousands of times, get the proportion of times that the criterion is met and/or that the complex event happened.

Cricket is..

Cricket is a discrete event game like baseball each ball thrown can result in 0, 1, 2, 3 (rare), 4, 5 (rare), or 6 runs or an out/wicket.

In the Twenty20 format, up to 120 balls (pitches) are thrown per side, as long as they are within the equivalent of a strike zone. The value of interest is the total runs scored before either 120 balls are thrown or 10 wickets/outs are taken.

Like baseball, there is a batsman (batter) and a bowler (pitcher), and a collection of fielders. There are only 2 bases and getting from one base to the other is a run.

Here’s our dataset. One row is one ball thrown. There are about 800 matches here, totalling over 160,000 balls thrown.

Our data

We can estimate the probabilities of these events (0,1,2,3,4,5,6,out) a variety of ways.

  • The global average rate for the league
  • The personal average rate for batters.
  • The average of batter and bowler rate.
  • Game situation dependent.

Example: If there are only a few balls left and many wickets remaining, then wickets, fours, and sixes become relatively more common.

Simulation

set.seed(8675309)
Nsims = 5000
allsims_runs = rep(NA, Nsims)
allsims_outs = rep(NA, Nsims)
allsims_runs_batter = matrix(NA, nrow=Nsims, ncol=10)

for(simcount in 1:Nsims)
{
  runs_this_batter = rep(0,10)
  runs = 0
  outs = 0
  balls_thrown = 0
  while(balls_thrown < 120 & outs < 10)
  {
  
      noball_chance = 0.0547
      outcome_chance = outcome_chance_naive
                      # W, 0, 1, 2, 3, 4, 5, 6
  
      if(runif(1) > noball_chance){balls_thrown = balls_thrown + 1}
  
      outcome_idx = sample(1:8, 1, prob=outcome_chance)
      runs = runs + c(0,0,1:6)[outcome_idx]
      outs = outs + c(1,0,0,0,0,0,0,0)[outcome_idx]
  
      if(outs < 10)
      {
          runs_this_batter[outs+1] = runs_this_batter[outs+1] + c(0,0,1:6)[outcome_idx]
      }
  }
  
allsims_runs[simcount] = runs
allsims_outs[simcount] = outs
allsims_runs_batter[simcount, ] = runs_this_batter
  
}

Now let’s modify our simulator to account for overs and see the output again.

set.seed(8675309)
Nsims = 5000
allsims_runs = rep(NA, Nsims)
allsims_outs = rep(NA, Nsims)
allsims_runs_batter = matrix(NA, nrow=Nsims, ncol=10)
allsims_runs_over = matrix(NA, nrow=Nsims, ncol=20)

for(simcount in 1:Nsims)
{
  runs_this_batter = rep(0,10)
  runs_this_over = rep(0,20)
  runs = 0
  outs = 0
  balls_thrown = 0
  while(balls_thrown < 120 & outs < 10)
  {
      over = ceiling((balls_thrown + 1) / 6)
      noball_chance = 0.0547
      #outcome_chance = outcome_chance_naive
      outcome_chance = outcome_chance_over[over,]
                      # W, 0, 1, 2, 3, 4, 5, 6
  
      if(runif(1) > noball_chance){balls_thrown = balls_thrown + 1}
  
      outcome_idx = sample(1:8, 1, prob=outcome_chance)
      runs = runs + c(0,0,1:6)[outcome_idx]
      outs = outs + c(1,0,0,0,0,0,0,0)[outcome_idx]
  
      if(outs < 10)
      {
          runs_this_batter[outs+1] = runs_this_batter[outs+1] + c(0,0,1:6)[outcome_idx]
      }
      
      runs_this_over[over] = runs_this_over[over] + c(0,0,1:6)[outcome_idx]
  }
  
allsims_runs[simcount] = runs
allsims_outs[simcount] = outs
allsims_runs_batter[simcount, ] = runs_this_batter
allsims_runs_over[simcount, ] = runs_this_over
  
}
